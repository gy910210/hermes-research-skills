# Monitoring Runbook

## 推荐用途
- 周期性跟踪一个研究方向的新论文、新 benchmark、新官方信号。
- 为成熟主稿维护“增量观察稿”或“待更新候选池”。

## 默认优先级
1. arXiv 新论文与技术报告
2. benchmark / dataset 官方页面更新
3. 官方产品、工程与研究博客
4. 会议 accepted/tutorial 页面

## 何时认为“值得更新知识库”
- 出现了与当前主线强相关的 `direct evidence`
- 出现了能修正旧结论的反例或矛盾证据
- 出现了新的 benchmark / dataset release / official launch
- 出现了新的校企结合案例，能明显改善“研究 vs 落地”这一节

## 何时只保留为观察记录
- 只出现同类论文的轻量增量
- 只是二手报道，没有原始来源
- 只有 headline，没有方法、数据、发布日期等硬信息

## 推荐输出结构
1. 本轮监测范围与时间窗口
2. 新来源清单
3. 值得更新的点
4. 暂不更新的点
5. 下一步建议

## 自动化提示建议
- prompt 里要写清：
  - 主题或已有文档路径
  - 监测时间窗口
  - 目标输出文件路径
  - 若无强更新则只写摘要，不改主稿
