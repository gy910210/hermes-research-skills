# Report Types

## 增量稿
- 适合：继续探索、补证据、扩展边界
- 结构：新增问题 -> 新证据 -> 方向判断 -> 可并回主稿部分

## 专题稿
- 适合：围绕单一子方向深挖
- 结构：问题定义 -> 近期论文 -> 关键矛盾 -> 建议回写主稿的小节

## 方向比较稿
- 适合：支持与矛盾、路线比较、方法对比
- 结构：比较轴 -> 支持证据 -> 冲突证据 -> 综合判断

## 双视角稿
- 适合：学术界 vs 工业界，或研究 vs 落地
- 结构：学术线 -> 工业线 -> 结合视角 -> 桥接缺口

## 成熟主稿
- 适合：阶段性正式交付
- 结构：问题定义 -> 演化 -> 系统分层 -> 主方向 -> 工业落地 -> 数据集/benchmark -> 风险与建议 -> 附录

## Awesome 页面
- 适合：维护 `awesome-*.md`、repo README、长列表资源页
- 结构：范围说明 -> 阅读地图 -> 条目分层 -> 最新高价值新增 -> 维护规则

## 机构路线图
- 适合：按快手/腾讯/美团/Meta-Pinterest 等机构或平台比较方法家族
- 结构：写作边界 -> 机构线 -> 横向对比表 -> route-level finding

## 多轮批判性迭代稿
- 适合：要求三轮以上 critical thinking、论文 idea 碰撞、稳定 finding 提炼
- 结构：边界 -> 技术栈/主题树 -> 多轮迭代 -> 稳定 finding -> 潜在问题 -> 下一轮问题

## 路线族比较稿
- 适合：比较不同机构路线或不同方法家族
- 结构：比较轴 -> 机构/路线族 -> 最强证据 -> 差异与短板 -> 综合判断
