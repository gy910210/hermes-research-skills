# Diagram Selection Guide

## 快速判断

- 选择结构图：原文在讲组成、层次、模块边界、能力栈、系统总图。
- 选择流程图：原文在讲步骤、阶段、pipeline、决策流、状态迁移。
- 选择时序图：原文在讲多角色之间谁先调用谁、谁响应谁、谁授权谁。
- 选择对比图：原文在讲 A/B 路径、学术/工业、优劣、矛盾、trade-off。
- 选择时间线：原文在讲 2024/2025/2026 这类时间推进或阶段演化。
- 选择矩阵图：原文在讲两个维度交叉，例如价值 x 风险、能力 x 可控性。

## 结构图 vs 流程图

- 结构图回答“由什么组成、怎么分层、谁依赖谁”。
- 流程图回答“先做什么、再做什么、在哪个节点分支或闭环”。
- 如果原文同时包含结构与流程，优先拆成两张图，不要塞进同一张图里。

## 时序图判断信号

出现下列信号时优先考虑时序图：

- `user -> agent -> tool -> platform` 这类调用链
- request / response / callback / authorize / verify / checkout
- 多角色在一轮任务里的往返交互

## 对比图判断信号

出现下列信号时优先考虑对比图：

- `路径 A / 路径 B`
- `学术界 / 工业界`
- `优点 / 风险 / 边界`
- `更适合`、`不适合`、`trade-off`

优先使用表格或双栏对比图；只有在关系明显呈对称结构时才画成左右镜像图。

## 不要画图的情况

- 原文只是松散罗列观点，没有稳定关系。
- 原文证据不足，图会迫使模型补全不存在的因果。
- 一张图需要超过 12 个节点才能说明白。
- 图不能比原文更快被理解。
